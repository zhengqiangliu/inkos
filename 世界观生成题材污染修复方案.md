# 世界观生成题材污染问题 - 根因分析与修复方案

## 问题现象
用户在创建书籍时输入"都市主题"，生成世界观时却产出了修仙内容（北境剑门、灵石、仙域等），与预期主题完全不符。

## 根因分析

### 1. **题材上下文传递失败** (主要根因)
**位置**：`packages/core/src/interaction/project-tools.ts:1009-1011`

```typescript
const genreContext = existingDraft?.genre && projectRoot
  ? await readGenreProfile(projectRoot, existingDraft.genre).catch(() => null)
  : null;
```

**问题**：
- 题材约束的读取完全依赖 `existingDraft?.genre`
- 当用户只输入"都市主题"但未更新 draft.genre 时，该字段可能：
  - 仍然是空值
  - 残留上一本书的值（如 "xianxia"）
  - 导致 `readGenreProfile` 读取了错误的题材规则或返回 null

### 2. **前端未传递题材信息**
**位置**：`packages/studio/src/pages/BookCreate.tsx:921-943`

```typescript
const generateWizardStepBody = useCallback(async (targetStep, instruction) => {
  const response = await sendMessage(activeSessionId, instruction, activeBookId ?? undefined, {
    skipAutoNewPrefix: true,
    wizardStep: targetStep,
    forceStream: true,
  });
  // 没有传递 themeGenre 或 genre 参数
}, [activeBookId, activeSessionId, ...]);
```

**问题**：
- 前端调用世界观生成时，仅传递了 `instruction` 和 `wizardStep`
- 没有像简介生成那样传递 `themeGenre` 参数
- 后端无法从请求中获取当前选择的题材

### 3. **Prompt 缺少题材约束兜底**
**位置**：`packages/core/src/interaction/project-tools.ts:865-866`

```typescript
const genreBlock = genreContext
  ? [/* 题材信息 */].join("\n")
  : "";  // genreContext 为 null 时完全不包含题材信息
```

**问题**：
- 当 `genreContext` 为 null 时，prompt 中完全没有题材约束
- LLM 可能基于：
  - 对话历史中的残留信息（如之前的修仙书籍）
  - 训练数据的偏好
  - 随机生成

## 完整修复方案

### 修复1：前端传递题材参数 (关键修复)

**文件**：`packages/studio/src/pages/BookCreate.tsx`

**位置**：在 `generateWizardStepBody` 函数中传递题材

```typescript
const generateWizardStepBody = useCallback(async (targetStep: Exclude<BookCreationWizardStep, "intro">, instruction: string): Promise<string> => {

  // 🔧 修复：增加题材上下文传递
```

**修改说明**：
- 从当前选中的题材 `selectedGenre?.id` 或 draft 中读取题材
- 优先级：`selectedGenre?.id` > `draft?.genre` > `draft?.mappedGenreId`
- 通过 `sendMessage` 的 options 传递给后端

### 修复3：Prompt 增加题材兜底约束

**文件**：`packages/core/src/interaction/project-tools.ts`

**位置**：`buildWizardPrompt` 函数（第796-888行）

```typescript
export function buildWizardPrompt(
  step: BookCreationWizardStep,
  mode: WizardMode,
  userMessage: string,
  existingDraft?: BookCreationDraft,
  genreContext?: WizardGenreContext,
  explicitGenre?: string,  // 🔧 新增：显式题材参数
): string {
  const template = step === "intro" && mode === "generate"
    ? INTRO_GENERATION_PROMPT
    : getWizardStepTemplate(step);
  const allowedFields = WIZARD_STEP_FIELDS[step].join("、");
  const draftBlock = existingDraft
    ? ["## 当前草案", JSON.stringify(existingDraft, null, 2)].join("\n")
    : "## 当前草案\n（空）";
  
  // ... stepContext 构建逻辑
  
  const modeLabel = step === "intro" && mode === "generate"
    ? "生成正式简介"
    : mode === "generate"
      ? "生成当前页"
      : "只修改当前页";
  
  const genreBlock = genreContext
    ? [/* 完整题材约束 */].join("\n")
    : explicitGenre  // 🔧 新增：即使没有 genreContext，也显式说明题材
      ? [
          "## 题材约束",
          `- 题材：${explicitGenre}`,
          "- 必须严格遵守该题材的风格、世界观特征和常见设定。",
          "- 禁止混入其他题材的专有名词、体系或世界规则。",
        ].join("\n")
      : "";

  return [
    `当前步骤：${template.title}`,
    `模式：${modeLabel}`,
    "",
    ...(genreBlock ? [genreBlock, ""] : []),
    ...(stepContext ? [stepContext, ""] : []),
    // ... 后续约束
  ].join("\n");
}
```

## 验证方案

### 测试用例1：新建书籍
1. 清空所有会话历史
2. 选择"都市"题材
3. 输入主题："现代职场竞争"
4. 生成世界观
5. **预期**：产出都市背景的公司、职位、商业规则等内容

### 测试用例2：切换题材
1. 先创建一本"修仙"题材的书
2. 返回首页，新建书籍
3. 选择"都市"题材
4. 生成世界观
5. **预期**：不包含任何修仙相关内容（宗门、灵石、剑修等）

### 测试用例3：无题材选择兜底
1. 不选择任何题材
2. 仅输入"科技公司竞争"
3. 生成世界观
4. **预期**：基于用户输入推断出都市/职场背景，不使用历史题材

## 优先级建议

1. **P0（立即修复）**：修复1 + 修复2 - 保证题材正确传递和使用
2. **P1（跟随修复）**：修复3 - prompt 兜底，防止边界情况
3. **P2（可选）**：修复4 - 类型完善，增强代码可维护性

## 影响范围

- **前端**：1个文件，1个函数修改
- **后端**：1个文件，3处修改（参数增加、逻辑调整、prompt 增强）
- **向下兼容**：新增参数为可选，不影响现有调用
- **测试回归**：需要重点测试世界观、大纲、卷纲、人物弧光、人物关系等所有非简介页的生成

## 根本原因总结

题材污染问题的本质是**题材上下文的单向依赖**：

```
用户选择题材 → 前端状态 → (❌ 未传递) → 后端调用 → (✓ 读取 draft.genre) → LLM
                                                    ↑
                                                可能是上一本书的值
```

修复后的流程：

```
用户选择题材 → 前端状态 → (✓ 显式传递) → 后端调用 → (✓ 优先使用请求值) → LLM
                                                    ↓
                                                兜底到 draft.genre
                                                    ↓
                                                prompt 强制约束
```

这样形成了三层防护，确保题材信息正确传递和使用。
